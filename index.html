Four Strategic Pillars

EVSI (Earliest Verifiable Source Index)

Every event includes snapshots + timestamps + evidence chains to auditably prove "we detected it first"

Transforms our speed advantage from marketing claim to verifiable metric. Creates competitive moat through data transparency—competitors can't easily replicate without similar evidence infrastructure.

IPM (Impact Pathway Model)

Event → Entity Graph (assets/upstream/regulatory) → Your specific risk exposure (impact score/direction)

Solves the "So What?" problem. Users don't want to know that something happened—they want to know how it affects their portfolio, compliance posture, or operational risk. This bridges the gap between information and action.

SCG + RDS (Source Credibility Graph + Risk Delta Scoring)

Source reputation mapping (reduce false positives) + Risk increment scoring (increase actionability)

SCG learns from user feedback to weight sources based on historical accuracy and relevance. RDS focuses attention on meaningful changes rather than noise. Together, they create a self-improving signal-to-noise ratio.

LPD (Latency-per-Dollar Optimization)

Shared trigger caching, two-tier decision making, priority queues for faster and cheaper operations

Speed advantage must be economically sustainable. Shared infrastructure reduces marginal costs while maintaining performance. Two-tier processing (fast screening + careful analysis) optimizes the speed/accuracy tradeoff for different alert priorities.

Trust-Building Quartet

  • Evidence First: Every alert comes with source links, diff snippets, discovery time, lead minutes, confidence score; one-click "verify at source"
  • Shadow Comparison: New users see "official source timeline vs our hits" for first 7-14 days
  • Public SLO Status Page: Real-time display of P50/P95 latency, effectiveness rate, noise rate + incident post-mortems
  • Feedback Loop: One-click labeling of "false positive/reason", system auto-converges thresholds and blacklist terms
Trust is the biggest barrier in risk intelligence—one missed critical event destroys credibility forever. Shadow comparison provides immediate proof of value, while public SLOs demonstrate confidence in our performance. Evidence-first design turns every alert into a defensible audit trail.

Anti-Anxiety (Calm UX)

  • Guardian mode (Critical only) + quiet hours + All-clear peace receipts + hourly/daily summaries
  • Configurable daily alert limits (Critical unlimited); non-stimulating colors, refresh cooldown
Risk professionals suffer from chronic alert fatigue and burnout. By designing for calm rather than attention-grabbing, we increase long-term retention and decision quality. All-clear receipts specifically address the "what if I missed something" anxiety that drives compulsive checking behavior.

Technology & Cost Architecture

Explainable, scalable, cost-controllable

  • Scraping Priority: API/RSS > HTTP (with ETag/Last-Modified conditional requests) > Headful rendering (Playwright when necessary)
  • Diff & Models: Only apply similarity pre-filtering to diff snippets → small model classification → minimal large model use (cost reduction)
  • Queues & Source Sharing: Adaptive frequency control based on source change rate and user SLA; same URL scraped once, shared across users
  • Observability: 304 hit rate, change rate, latency, AI cost per effective alert, false positive complaint rate
This architecture prioritizes marginal cost reduction over infrastructure complexity. API/RSS sources cost $0.001/check vs $0.1/headful render. Diff-only processing reduces LLM token usage by 95%. Shared collection with per-user intelligence creates natural economies of scale—more users = lower per-user costs.

Anti-Scraping Strategy

  • Network/CDN Layer: IP/ASN rate limiting, reputation scoring, seamless challenges; search engine whitelist
  • Application Layer: HMAC signed parameters, cursor pagination, max depth/response body limits, repeat scraping 304
  • Frontend Layer: Light DOM polymorphism, honeypots/watermarks (forensics, no SEO/accessibility impact)
  • Identity & Audit: Account/organization quotas, behavior scoring, evidence chain
  • Front Door: Official API for partners (signing, quotas, complexity limits, audit)
Anti-scraping isn't about blocking legitimate use—it's about sustainable resource allocation. Rate limiting prevents infrastructure overload, while honeypots identify bad actors. The "front door" API creates a revenue stream from heavy users while reducing adversarial scraping pressure.

Commercialization & Packaging

Tiered Package Strategy

Three-tier packages (value-oriented, not technical feature stacking):

  • Pro Tier: Individual professionals and small teams - focus on personal productivity gains
  • Team Tier: Growing organizations with collaboration needs - shared workflows and audit trails
  • Enterprise Tier: Large organizations requiring SSO/RBAC/audit export/data residency/dedicated proxy pools/custom sources/SLA
Tiering by organizational complexity, not usage volume, aligns pricing with value delivered. Pro users pay for personal time savings, Team users pay for coordination benefits, Enterprise pays for compliance and integration. This creates natural upgrade paths as organizations grow.

Sales Materials: Cases presented in "lead minutes → action → avoided losses" structure; technical depth relegated to security whitepapers and procurement attachments.

Decision makers care about ROI, not technical sophistication. A single case study of "our alert 15 minutes before official announcement prevented $2M trading loss" is worth more than any feature list. Technical details become relevant only after value is established.
